# IDS 模块评估报告：RoboRefit 歧义消解实验

> **实验日期**: 2026-05-16  
> **模型**: `IffYuan/Embodied-R1-3B-v1`  
> **数据集**: RoboRefit test set（2000 样本）  
> **相关代码**: `disambiguation/`、`experiments/analyze_ids_roborefit.py`、`eval/hf_inference_roborefit_degrade.py`

---

## 一、背景与目标

IDS（Instruction Disambiguation System）是一个轻量级指令歧义消解模块，设计目标是在机器人执行指令之前，识别存在歧义的操控指令（如场景中有多个同类物体时），并主动询问用户澄清，而非让模型盲目猜测。

本次评估的核心问题是：**IDS 的消歧能力是否真正提升了系统在模糊指令上的执行准确率？**

由于 IDS 的价值不在于提升明确指令上的模型精度，而在于将"盲目执行歧义指令"转化为"询问用户后再执行具体指令"，因此需要构造受控实验来量化这一提升。

---

## 二、实验设计

### 2.1 为什么不能直接用 eval baseline 对比

初步分析发现，直接在 RoboRefit eval baseline 上运行 IDS 并不能得到有意义的对比：

- eval 数据集中每条指令都有确定的 ground truth，设计上无歧义
- IDS 文本启发式（无 `MCPStateTracker`）只能检测 `_GENERIC_OBJECTS` 词表中无属性词的指令
- 实验结果：IDS 标记为 ambiguous 的 244 条准确率反而高于 resolved 的 1756 条（90.2% vs 84.7%），说明文本启发式在无场景状态的情况下误判严重

### 2.2 受控实验方案：指令退化对比

在同一张图上构造两个条件：

| 条件 | 指令形式 | 示例 |
|------|---------|------|
| 具体指令（上界） | 保留属性词 | `"pick up the yellow glue stick"` |
| 退化指令（歧义模拟） | 去除颜色/尺寸/空间词 | `"pick up the glue stick"` |

**关键约束**：只有退化后 IDS 能检测为 ambiguous、且原始指令 IDS 判为 resolved 的样本才纳入实验。这确保了"IDS 本可拦截该歧义"这一前提成立。

### 2.3 三条对比线

```
Upper bound  ：全部用具体指令（baseline 已知）
New baseline ：全部用退化指令跑 VLM（模拟无 IDS 时的真实水平）
With IDS     ：IDS 能检测的样本换回具体指令；检测不到的维持退化指令
```

**With IDS** 模拟了真实部署场景：IDS 检测到歧义 → 询问用户 → 用户补充属性词 → 模型执行具体指令。

## 三、实验结果

### 3.1 样本筛选

| 类别 | 数量 |
|------|------|
| RoboRefit 全量样本 | 2000 |
| 含属性词、可退化的样本 | 1428（71.4%）|
| 退化后 IDS 能检测到的样本 | **350**（24.5% of 1428）|
| 退化后 IDS 也检测不到的样本 | 1078（75.5% of 1428）|

### 3.2 主要指标

| 条件 | 准确率 |
|------|--------|
| **Upper bound**（具体指令） | **0.8487** |
| **New baseline**（退化指令，无 IDS） | **0.5686** |
| **With IDS**（检测到→具体，未检测到→退化） | **0.5998** |
| **IDS 真实提升**（vs new baseline） | **+0.0312** |
| 距 upper bound 的剩余 gap | 0.2489 |

### 3.3 IDS 覆盖子集的详细对比

| 子集 | 样本数 | 退化指令准确率 | 具体指令准确率 | 提升 |
|------|--------|--------------|--------------|------|
| IDS detectable | 350 | 0.7400 | **0.8671** | **+0.1271（+17.2%）** |
| IDS undetectable | 1078 | 0.5130 | — | IDS 无法介入 |

## 四、结果分析

### 4.1 IDS 消歧有效性得到验证

在 IDS 能检测到的 350 条样本上，将退化指令换回具体指令后，准确率从 0.74 提升至 0.87，**绝对提升 12.7 个百分点**。这直接证明了 IDS 的消歧机制有效：通过询问用户补全属性词，VLM 的执行准确率显著恢复。

### 4.2 整体提升受覆盖率制约

在全部 1428 条退化样本上，IDS 整体提升仅 +0.03，原因是 IDS 当前只能覆盖 24.5% 的样本。剩余 75.5%（1078 条）IDS 检测不到的样本保持退化指令，拉低了整体数字。

覆盖率不足的根本原因是 `_GENERIC_OBJECTS` 词表过窄，无法覆盖 RoboRefit 的物体词汇（如 "glue stick"、"screwdriver"、"thermometer" 等均不在词表中）。

### 4.3 退化函数的数据质量问题

当前 `degrade_instruction` 实现为逐词删除空间词，导致部分退化指令出现悬挂介词：

```
原始: "pick up the screwdriver on the right"
退化: "pick up the screwdriver on the"   ← "on the" 悬挂
```

这会影响实验的干净程度，但由于模型对此类指令有一定鲁棒性，未显著影响结论的方向。后续应改进为正则替换整个短语（`"on the right"` → 整体去除）。

---

## 五、局限性

1. **IDS 覆盖率低（24.5%）**：核心原因是仅依赖文本启发式，无 `MCPStateTracker` 场景状态支持。在真实部署中，若有结构化 object state 输入，覆盖率会大幅提升。

2. **澄清模拟为 oracle**：实验用原始具体指令替代"用户回答澄清问题后的指令"，这是理想情况。真实场景中用户的回答未必能完全恢复到原始指令形式。

3. **单一数据集**：仅在 RoboRefit 上验证，其他适用数据集（VABench、Open6DoR）尚未做对应实验。

---

## 六、结论

| 结论 | 支持证据 |
|------|---------|
| IDS 消歧机制有效 | detectable 子集准确率提升 +12.7% |
| 当前文本启发式覆盖率不足 | 仅覆盖 24.5% 的样本 |
| 扩充 `_GENERIC_OBJECTS` 词表是最直接的改进方向 | 1078 条未覆盖样本中大量含 RoboRefit 特有物体名词 |
| 无场景状态时 IDS 不宜直接用于 eval benchmark 对比 | 文本启发式在无 MCPStateTracker 时误判率高 |

---

## 七、后续建议

1. **扩充 `_GENERIC_OBJECTS`**：加入 RoboRefit/VABench 高频物体名词，估计可将覆盖率从 24.5% 提升至 50%+
2. **改进退化函数**：用短语级正则替换，避免悬挂介词
3. **接入 MCPStateTracker**：在有结构化场景状态的数据集（VABench、Open6DoR）上重复本实验，预期覆盖率和提升幅度均会更高
4. **扩展到 VABench 和 Open6DoR**：这两个数据集的指令更贴近 IDS 设计场景，open6dor 已发现 IDS 能正确识别 "Place the box behind the box" 类真实歧义
